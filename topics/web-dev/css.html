CSS - Cascading Style Sheets

CSS is the language used to style and layout web pages. It controls colors, fonts, spacing, positioning, and overall visual presentation of HTML elements.

CSS Introduction

CSS (Cascading Style Sheets) is a stylesheet language that describes how HTML elements should be displayed on screen, paper, or other media.

What is CSS?

  • CSS = Cascading Style Sheets
  • Controls the visual presentation of HTML documents
  • Separates content (HTML) from presentation (CSS)
  • Makes websites responsive and visually appealing
  • Can control multiple web pages at once

Why Use CSS?

  • Separation of Concerns: Keep HTML for structure, CSS for styling
  • Reusability: One CSS file can style multiple pages
  • Maintainability: Easy to update styles across entire website
  • Faster Loading: CSS files are cached by browsers
  • Better UX: Create visually appealing, professional websites

Adding Styles Using CSS

There are three ways to add CSS to HTML documents:

1. Inline CSS

CSS applied directly to HTML elements using the style attribute.

<p style="color: blue; font-size: 18px;">This is a paragraph</p>

Pros:

  • Quick for testing
  • Highest priority (overrides other styles)

Cons:

  • Hard to maintain
  • Not reusable
  • ❌ Not recommended for production

2. Internal CSS (Embedded)

CSS written in the <style> tag within the <head> section.

<head>
  <style>
    p {
      color: blue;
      font-size: 18px;
    }
  </style>
</head>

Pros:

  • Good for single-page styling
  • No extra HTTP request

Cons:

  • Not reusable across pages
  • Makes HTML file larger

3. External CSS ✅ (Best Practice)

CSS written in a separate .css file and linked to HTML.

In HTML file:

<head>
  <link rel="stylesheet" href="style.css">
</head>

In style.css file:

p {
  color: blue;
  font-size: 18px;
}

Pros:

  • ✅ Best for multiple pages
  • ✅ Easy to maintain
  • ✅ Reusable
  • ✅ Browser caching
  • ✅ Clean HTML

CSS Syntax

selector {
  property: value;
  property: value;
}
  • Selector: Points to the HTML element you want to style
  • Property: The aspect you want to change (color, font-size, etc.)
  • Value: The setting for the property
  • Semicolon (;): Separates declarations
  • Curly Braces {}: Contains all declarations

CSS Selector Types

Selectors are patterns used to select and style HTML elements.

1. Element Selector

Selects all elements of a given type.

p {
  color: red;
}

/* Selects all <p> elements */

2. Class Selector

Selects elements with a specific class attribute. Uses a period (.).

.highlight {
  background-color: yellow;
}

/* Selects all elements with class="highlight" */

HTML: <p class="highlight">Text</p>

3. ID Selector

Selects a single element with a specific id attribute. Uses a hash (#).

#header {
  font-size: 24px;
}

/* Selects element with id="header" */

HTML: <div id="header">Header</div>

4. Universal Selector

Selects all elements. Uses an asterisk (*).

* {
  margin: 0;
  padding: 0;
}

/* Selects all elements */

5. Attribute Selector

Selects elements based on attributes.

input[type="text"] {
  border: 1px solid blue;
}

/* Selects all text input fields */

Grouping and Nesting of CSS Selectors

Grouping Selectors

Apply the same style to multiple selectors by separating them with commas.

h1, h2, h3 {
  color: navy;
  font-family: Arial;
}

/* Applies to all h1, h2, and h3 elements */

Nesting Selectors (Descendant Selector)

Select elements inside other elements.

div p {
  color: blue;
}

/* Selects all <p> inside <div> */

Other Combinators:

  • Child Selector (>): div > p - Direct children only
  • Adjacent Sibling (+): h1 + p - Immediately after
  • General Sibling (~): h1 ~ p - All siblings after

Specificity of Selectors

When multiple CSS rules target the same element, specificity determines which style is applied.

Specificity Hierarchy (Highest to Lowest):

1. Inline Styles

style="color: red;" - Specificity: 1000

2. IDs

#header - Specificity: 100

3. Classes, Attributes, Pseudo-classes

.button, [type="text"], :hover - Specificity: 10

4. Elements, Pseudo-elements

p, ::before - Specificity: 1

Calculating Specificity Example:

p { color: red; } /* Specificity: 1 */

.intro { color: blue; } /* Specificity: 10 */

#main p { color: green; } /* Specificity: 101 */

/* #main p wins! (101 > 10 > 1) */

Preference of Rules (The Cascade)

CSS stands for "Cascading" Style Sheets. When multiple rules apply to an element, these factors determine which style wins:

Order of Importance:

  1. !important - Overrides everything (use sparingly!)
  2. Inline styles - style attribute in HTML
  3. Specificity - More specific selectors win
  4. Source order - Later rules override earlier ones
p {
  color: blue;
}

p {
  color: red;
}

/* Paragraph will be red (last rule wins) */

How to Open DevTools:

  • Right-click on any element → "Inspect"
  • Keyboard shortcuts:
    • Windows/Linux: F12 or Ctrl + Shift + I
    • Mac: Cmd + Option + I

Adding Color

Colors in CSS can be specified in several ways:

Color Formats:

1. Color Names

color: red;
background-color: blue;
/* 140 named colors like red, blue, green, etc. */

2. Hexadecimal (Hex)

color: #FF0000; /* Red */
color: #00FF00; /* Green */
color: #0000FF; /* Blue */
color: #FFF; /* Short for #FFFFFF (white) */

3. RGB (Red, Green, Blue)

color: rgb(255, 0, 0); /* Red */
color: rgb(0, 255, 0); /* Green */
/* Values: 0-255 for each channel */

4. RGBA (RGB with Alpha/Opacity)

color: rgba(255, 0, 0, 0.5); /* 50% transparent red */
/* Alpha: 0 (transparent) to 1 (opaque) */

5. HSL (Hue, Saturation, Lightness)

color: hsl(0, 100%, 50%); /* Red */
color: hsla(120, 100%, 50%, 0.3); /* 30% transparent green */

CSS Units

CSS units specify the measurement of properties like width, height, font-size, etc.

Absolute Units (Fixed Size)

Unit Description
px Pixels - Most common, 1px = 1/96th of an inch
pt Points - 1pt = 1/72nd of an inch
cm, mm, in Centimeters, millimeters, inches

Relative Units (Responsive)

Unit Description
% Percentage - Relative to parent element
em Relative to font-size of the element (2em = 2 times)
rem Relative to root (html) font-size (more predictable)
vw Viewport width - 1vw = 1% of viewport width
vh Viewport height - 1vh = 1% of viewport height

The CSS Box Model

The Box Model is fundamental to CSS layout. Every HTML element is treated as a rectangular box.

Box Model Components (from inside out):

CONTENT
Padding
Border
Margin
  • Content: The actual content (text, images)
  • Padding: Space between content and border
  • Border: Line around padding
  • Margin: Space outside the border

Box-Sizing Property:

/* Default - padding and border add to width */
box-sizing: content-box;

/* Better - padding and border included in width */
box-sizing: border-box; /* ✅ Recommended */

Margins

Margins create space OUTSIDE an element's border, between elements.

Margin Properties:

/* Individual sides */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 10px;
margin-left: 20px;

/* Shorthand - 4 values (top, right, bottom, left) */
margin: 10px 20px 10px 20px;

/* Shorthand - 2 values (vertical, horizontal) */
margin: 10px 20px;

/* Shorthand - 1 value (all sides) */
margin: 10px;

/* Center element horizontally */
margin: 0 auto;

/* Negative margin (pull element) */
margin-top: -10px;

⚠️ Margin Collapse

When two vertical margins meet, they collapse into a single margin (the larger one). This doesn't happen with padding!

Padding

Padding creates space INSIDE an element's border, between content and border.

Padding Properties:

/* Individual sides */
padding-top: 10px;
padding-right: 20px;
padding-bottom: 10px;
padding-left: 20px;

/* Shorthand - 4 values (top, right, bottom, left) */
padding: 10px 20px 10px 20px;

/* Shorthand - 2 values (vertical, horizontal) */
padding: 10px 20px;

/* Shorthand - 1 value (all sides) */
padding: 10px;

Margin vs Padding:

Aspect Margin Padding
Location Outside border Inside border
Background No background color Has background color
Collapse Can collapse Never collapses
Negative values ✅ Allowed ❌ Not allowed

Display Property

The display property controls how an element is displayed and how it interacts with other elements.

Display Values:

block

display: block;
  • Takes full width available
  • Starts on a new line
  • Can set width and height
  • Examples: <div>, <p>, <h1>

inline

display: inline;
  • Only takes space needed
  • Stays on same line
  • Cannot set width/height
  • Examples: <span>, <a>, <strong>

inline-block

display: inline-block;
  • Stays on same line (like inline)
  • Can set width/height (like block)
  • Best of both worlds!

none

display: none;
  • Element is completely hidden
  • Takes no space
  • Different from visibility: hidden (which takes space)

flex

display: flex;
  • Creates flexible layout container
  • Modern layout system
  • Aligns and distributes space easily

grid

display: grid;
  • Creates grid layout container
  • 2D layout system (rows and columns)
  • Powerful for complex layouts

Position Property

The position property specifies how an element is positioned in the document.

Position Values:

static (default)

position: static;

Normal document flow. top, right, bottom, left properties have no effect.

relative

position: relative;
top: 10px;
left: 20px;

Positioned relative to its normal position. Original space is preserved.

absolute

position: absolute;
top: 0;
right: 0;

Positioned relative to nearest positioned ancestor. Removed from normal flow.

fixed

position: fixed;
top: 0;
left: 0;

Positioned relative to viewport. Stays in place when scrolling. Great for navbars!

sticky

position: sticky;
top: 0;

Hybrid of relative and fixed. Sticks when you scroll past it.

Z-Index (Layering):

z-index: 10; /* Higher values appear on top */
/* Only works with positioned elements (not static) */

Min/Max Width and Height

Control the minimum and maximum dimensions of elements for responsive design.

Width Properties:

width: 300px; /* Fixed width */
width: 50%; /* Relative to parent */
max-width: 800px; /* Won't grow beyond this */
min-width: 200px; /* Won't shrink below this */

Height Properties:

height: 200px; /* Fixed height */
height: 100vh; /* Full viewport height */
max-height: 500px; /* Won't grow beyond this */
min-height: 100px; /* Won't shrink below this */

💡 Pro Tip:

Use max-width instead of width for responsive layouts. The element will shrink on smaller screens!

Responsive Container Example:

.container {
  width: 100%;
  max-width: 1200px; /* Don't exceed 1200px */
  min-width: 320px; /* Minimum mobile width */
  margin: 0 auto; /* Center it */
  padding: 0 20px;
}

Pseudo Classes & Pseudo Elements

Pseudo-classes and pseudo-elements let you style elements based on their state or position without adding extra HTML.

Link Related Pseudo Classes

Link States:

a:link { color: blue; } /* Unvisited link */
a:visited { color: purple; } /* Visited link */
a:hover { color: red; } /* Mouse over */
a:active { color: orange; } /* Being clicked */

⚠️ Order Matters! (LVHA)

Remember: Link → Visited → Hover → Active

Common Pseudo Classes

button:hover { background: blue; } /* On mouse over */

input:focus { border: 2px solid green; } /* When focused */

li:first-child { font-weight: bold; } /* First item */

li:last-child { color: red; } /* Last item */

li:nth-child(2) { color: blue; } /* 2nd item */

li:nth-child(odd) { background: #f0f0f0; } /* Odd items */

div:not(.special) { opacity: 0.5; } /* Not matching */

Pseudo Elements (::before & ::after)

Pseudo-elements create virtual elements that don't exist in the HTML.

::before

Inserts content BEFORE an element's content.

.quote::before {
  content: "❝ "; /* Required */

  font-size: 2rem;
}

❝ This is a quote with ::before

::after

Inserts content AFTER an element's content.

.external-link::after {
  content: " ↗";
  color: #2196f3;
}
External Link ↗

  • Decorative elements
  • Icons without extra HTML
  • Clearfix for floats

Responsive Web Design

Responsive design ensures your website looks good on all devices - from phones to desktops.

Why Responsive Design?

  • 📱 55%+ of web traffic is mobile
  • ✅ Better user experience across all devices
  • 🚀 Improved SEO rankings
  • 💰 Cost-effective (one site for all devices)
  • 📈 Higher conversion rates

Different Ways to Make Responsive

1. Flexible Grid Layouts

Use percentage-based widths instead of fixed pixels

2. Flexible Images

max-width: 100%; height: auto;

3. Media Queries

Apply different styles for different screen sizes

4. Flexbox & Grid

Modern layout systems that adapt automatically

5. Viewport Units

vw, vh, vmin, vmax scale with viewport size

Viewport Meta Tag

Essential for responsive design! Add this to your HTML <head>:

<meta name="viewport" content="width=device-width, initial-scale=1.0">
  • width=device-width: Match screen width
  • initial-scale=1.0: Initial zoom level

Media Queries

Apply different styles based on device characteristics (screen size, orientation, etc.).

Basic Syntax:

/* Mobile first - base styles */
.container {
  width: 100%;
  padding: 1rem;
}

/* Tablets and up */
@media (min-width: 768px) {
  .container {
    width: 750px;
    margin: 0 auto;
  }
}

/* Desktops and up */
@media (min-width: 1024px) {
  .container {
    width: 1200px;
  }
}

Common Breakpoints:

Device Min Width Icon
Mobile 0px - 767px 📱
Tablet 768px+ 📱
Desktop 1024px+ 💻
Large Desktop 1440px+ 🖥️

Desktop First vs Mobile First

🖥️ Desktop First

Start with desktop styles, then use max-width to adapt down.

/* Desktop styles */
.container { width: 1200px; }

/* Tablet */
@media (max-width: 1024px) {
  .container { width: 100%; }
}

❌ Less common today

📱 Mobile First ✅

Start with mobile styles, then use min-width to enhance up.

/* Mobile styles */
.container { width: 100%; }

/* Desktop */
@media (min-width: 1024px) {
  .container { width: 1200px; }
}

✅ Recommended approach

💡 Why Mobile First?

  • Mobile traffic is growing (55%+ of web traffic)
  • Easier to enhance than to strip down
  • Better performance (smaller initial CSS)
  • Forces focus on essential content

🎬 Animations in CSS

CSS animations allow you to create smooth, engaging motion effects without JavaScript. They can transform static pages into dynamic, interactive experiences.

Transitions and Transforms

CSS Transitions

Transitions allow you to change property values smoothly over a specified duration.

Syntax:

transition: property duration timing-function delay;

Example:

.button {
    background-color: #64b5f6;
    padding: 10px 20px;
    transition: all 0.3s ease-in-out;
    /* Property: all, Duration: 0.3s, Timing: ease-in-out */
}

.button:hover {
    background-color: #1976d2;
    transform: scale(1.1);
}

Live Demo:

Hover Me!

Transition Properties

Property Description Example Values
transition-property Which property to animate all, background, transform
transition-duration How long the animation takes 0.3s, 500ms, 2s
transition-timing-function Speed curve of the transition ease, linear, ease-in-out
transition-delay Delay before starting 0s, 0.5s, 1s

CSS Transforms

Transforms allow you to rotate, scale, skew, or translate an element.

/* Translate (Move) */
transform: translate(50px, 100px);  /* Move right 50px, down 100px */
transform: translateX(50px);        /* Move only horizontally */
transform: translateY(50px);        /* Move only vertically */

/* Scale (Resize) */
transform: scale(1.5);              /* Increase size by 150% */
transform: scale(0.5);              /* Decrease size by 50% */
transform: scaleX(2);               /* Scale only width */

/* Rotate */
transform: rotate(45deg);           /* Rotate 45 degrees clockwise */
transform: rotate(-90deg);          /* Rotate 90 degrees counterclockwise */

/* Skew (Slant) */
transform: skewX(20deg);            /* Slant horizontally */
transform: skewY(20deg);            /* Slant vertically */

/* Combine Multiple Transforms */
transform: translate(50px, 0) rotate(45deg) scale(1.2);

Creating a Simple Animation with @keyframes

The @keyframes rule specifies the animation code. You define what styles the element will have at certain times.

Basic Structure:

@keyframes animation-name {
    from { /* Starting styles */ }
    to { /* Ending styles */ }
}

Example 1: Simple Fade In

/* Define the animation */
@keyframes fadeIn {
    from {
        opacity: 0;
    }
    to {
        opacity: 1;
    }
}

/* Apply the animation */
.fade-in-element {
    animation: fadeIn 2s ease-in;
}

Example 2: Slide In from Left

@keyframes slideIn {
    0% {
        transform: translateX(-100%);
        opacity: 0;
    }
    100% {
        transform: translateX(0);
        opacity: 1;
    }
}

.slide-in-element {
    animation: slideIn 1s ease-out;
}

Example 3: Multiple Keyframes

@keyframes bounce {
    0% {
        transform: translateY(0);
    }
    25% {
        transform: translateY(-50px);
    }
    50% {
        transform: translateY(0);
    }
    75% {
        transform: translateY(-25px);
    }
    100% {
        transform: translateY(0);
    }
}

.bouncing-ball {
    animation: bounce 2s ease-in-out infinite;
}

Animation Properties

CSS animations have several properties to control their behavior:

Property Description Example Values
animation-name Name of @keyframes fadeIn, slideUp
animation-duration How long animation takes 2s, 500ms
animation-timing-function Speed curve ease, linear, ease-in-out
animation-delay Delay before starting 0s, 1s, 500ms
animation-iteration-count How many times to play 1, 3, infinite
animation-direction Direction of animation normal, reverse, alternate
animation-fill-mode Style before/after animation none, forwards, backwards, both
animation-play-state Pause or play animation running, paused

💡 Shorthand Syntax:

animation: name duration timing-function delay iteration-count direction fill-mode;

/* Example */
animation: slideIn 2s ease-in-out 0.5s infinite alternate forwards;

Detailed Example:

@keyframes pulse {
    0%, 100% {
        transform: scale(1);
        opacity: 1;
    }
    50% {
        transform: scale(1.1);
        opacity: 0.8;
    }
}

.pulsing-button {
    animation-name: pulse;
    animation-duration: 1.5s;
    animation-timing-function: ease-in-out;
    animation-delay: 0s;
    animation-iteration-count: infinite;
    animation-direction: normal;
    animation-fill-mode: none;
}

/* Or use shorthand */
.pulsing-button {
    animation: pulse 1.5s ease-in-out 0s infinite normal none;
}

Multiple Animations

You can apply multiple animations to a single element by separating them with commas.

Example: Combining Fade In + Slide Up

@keyframes fadeIn {
    from { opacity: 0; }
    to { opacity: 1; }
}

@keyframes slideUp {
    from { transform: translateY(30px); }
    to { transform: translateY(0); }
}

.animated-box {
    /* Apply both animations at once */
    animation: fadeIn 1s ease-out, slideUp 1s ease-out;
}

Example: Different Durations and Delays

@keyframes rotate {
    from { transform: rotate(0deg); }
    to { transform: rotate(360deg); }
}

@keyframes colorChange {
    0% { background-color: #e91e63; }
    50% { background-color: #9c27b0; }
    100% { background-color: #3f51b5; }
}

Starting with 3D Cube Animation

3D transformations in CSS allow you to create depth and perspective. Let's build a rotating 3D cube step by step.

Understanding 3D Transforms

Key 3D Properties:

  • perspective - Defines how far the element is from the viewer
  • transform-style: preserve-3d - Enables 3D space for children
  • rotateX(), rotateY(), rotateZ() - Rotate in 3D space
  • translateZ() - Move towards/away from viewer

Step 1: Create the HTML Structure

<div class="cube-container">
    <div class="cube">
        <div class="face front">Front</div>
        <div class="face back">Back</div>
        <div class="face right">Right</div>
        <div class="face left">Left</div>
        <div class="face top">Top</div>
        <div class="face bottom">Bottom</div>
    </div>
</div>

Step 2: Setup Container and Perspective

.cube-container {
    width: 200px;
    height: 200px;
    perspective: 1000px;  /* Creates 3D space */
    margin: 50px auto;
}

.cube {
    width: 100%;
    height: 100%;
    position: relative;
    transform-style: preserve-3d;  /* Enable 3D for children */
    animation: rotateCube 10s infinite linear;
}

@keyframes rotateCube {
    from {
        transform: rotateX(0deg) rotateY(0deg);
    }
    to {
        transform: rotateX(360deg) rotateY(360deg);
    }
}

Building the First Face of the Cube

Style the Face Elements:

.face {
    position: absolute;
    width: 200px;
    height: 200px;
    border: 2px solid #64b5f6;
    display: flex;
    align-items: center;
    justify-content: center;
    font-size: 24px;
    font-weight: bold;
    opacity: 0.9;
}

/* Front Face - No rotation needed */
.front {
    background: rgba(100, 181, 246, 0.7);
    transform: translateZ(100px);  /* Move forward by half the cube size */
}

⚠️ Important: translateZ(100px) pushes the face forward. Use half of your cube's width/height.

Building the Second Face of the Cube

Now let's add the back face. It needs to be rotated 180° and pushed back.

Back Face CSS:

.back {
    background: rgba(255, 152, 0, 0.7);
    transform: rotateY(180deg) translateZ(100px);
    /* 
       1. Rotate 180° around Y-axis (flip horizontally)
       2. Then move forward (which is now backward due to rotation)
    */
}

Complete All Six Faces

All Faces Together:

/* Front Face */
.front {
    background: rgba(100, 181, 246, 0.7);
    transform: translateZ(100px);
}

/* Back Face */
.back {
    background: rgba(255, 152, 0, 0.7);
    transform: rotateY(180deg) translateZ(100px);
}

/* Right Face */
.right {
    background: rgba(156, 39, 176, 0.7);
    transform: rotateY(90deg) translateZ(100px);
}

/* Left Face */
.left {
    background: rgba(233, 30, 99, 0.7);
    transform: rotateY(-90deg) translateZ(100px);
}

/* Top Face */
.top {
    background: rgba(76, 175, 80, 0.7);
    transform: rotateX(90deg) translateZ(100px);
}

/* Bottom Face */
.bottom {
    background: rgba(244, 67, 54, 0.7);
    transform: rotateX(-90deg) translateZ(100px);
}

💡 Pro Tips for 3D Animations:

  • Use perspective on the parent container, not the rotating element
  • Always set transform-style: preserve-3d on the parent of 3D children
  • Apply translateZ() AFTER rotation for correct positioning
  • Use backface-visibility: hidden to hide back faces when not needed
  • Higher perspective values (1500px-2000px) = less dramatic, more subtle 3D effect
  • Lower perspective values (500px-800px) = more dramatic, exaggerated 3D effect
