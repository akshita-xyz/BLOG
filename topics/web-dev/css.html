CSS - Cascading Style Sheets

CSS is the language used to style and layout web pages. It controls colors, fonts, spacing, positioning, and overall visual presentation of HTML elements.

CSS Introduction

CSS (Cascading Style Sheets) is a stylesheet language that describes how HTML elements should be displayed on screen, paper, or other media.

What is CSS?

  • CSS = Cascading Style Sheets
  • Controls the visual presentation of HTML documents
  • Separates content (HTML) from presentation (CSS)
  • Makes websites responsive and visually appealing
  • Can control multiple web pages at once

Why Use CSS?

  • Separation of Concerns: Keep HTML for structure, CSS for styling
  • Reusability: One CSS file can style multiple pages
  • Maintainability: Easy to update styles across entire website
  • Faster Loading: CSS files are cached by browsers
  • Better UX: Create visually appealing, professional websites

Adding Styles Using CSS

There are three ways to add CSS to HTML documents:

1. Inline CSS

CSS applied directly to HTML elements using the style attribute.

<p style="color: blue; font-size: 18px;">This is a paragraph</p>

Pros:

  • Quick for testing
  • Highest priority (overrides other styles)

Cons:

  • Hard to maintain
  • Not reusable
  • ❌ Not recommended for production

2. Internal CSS (Embedded)

CSS written in the <style> tag within the <head> section.

<head>
  <style>
    p {
      color: blue;
      font-size: 18px;
    }
  </style>
</head>

Pros:

  • Good for single-page styling
  • No extra HTTP request

Cons:

  • Not reusable across pages
  • Makes HTML file larger

3. External CSS ✅ (Best Practice)

CSS written in a separate .css file and linked to HTML.

In HTML file:

<head>
  <link rel="stylesheet" href="style.css">
</head>

In style.css file:

p {
  color: blue;
  font-size: 18px;
}

Pros:

  • ✅ Best for multiple pages
  • ✅ Easy to maintain
  • ✅ Reusable
  • ✅ Browser caching
  • ✅ Clean HTML

CSS Syntax

selector {
  property: value;
  property: value;
}
  • Selector: Points to the HTML element you want to style
  • Property: The aspect you want to change (color, font-size, etc.)
  • Value: The setting for the property
  • Semicolon (;): Separates declarations
  • Curly Braces {}: Contains all declarations

CSS Selector Types

Selectors are patterns used to select and style HTML elements.

1. Element Selector

Selects all elements of a given type.

p {
  color: red;
}

/* Selects all <p> elements */

2. Class Selector

Selects elements with a specific class attribute. Uses a period (.).

.highlight {
  background-color: yellow;
}

/* Selects all elements with class="highlight" */

HTML: <p class="highlight">Text</p>

3. ID Selector

Selects a single element with a specific id attribute. Uses a hash (#).

#header {
  font-size: 24px;
}

/* Selects element with id="header" */

HTML: <div id="header">Header</div>

4. Universal Selector

Selects all elements. Uses an asterisk (*).

* {
  margin: 0;
  padding: 0;
}

/* Selects all elements */

5. Attribute Selector

Selects elements based on attributes.

input[type="text"] {
  border: 1px solid blue;
}

/* Selects all text input fields */

Grouping and Nesting of CSS Selectors

Grouping Selectors

Apply the same style to multiple selectors by separating them with commas.

h1, h2, h3 {
  color: navy;
  font-family: Arial;
}

/* Applies to all h1, h2, and h3 elements */

Nesting Selectors (Descendant Selector)

Select elements inside other elements.

div p {
  color: blue;
}

/* Selects all <p> inside <div> */

Other Combinators:

  • Child Selector (>): div > p - Direct children only
  • Adjacent Sibling (+): h1 + p - Immediately after
  • General Sibling (~): h1 ~ p - All siblings after

Specificity of Selectors

When multiple CSS rules target the same element, specificity determines which style is applied.

Specificity Hierarchy (Highest to Lowest):

1. Inline Styles

style="color: red;" - Specificity: 1000

2. IDs

#header - Specificity: 100

3. Classes, Attributes, Pseudo-classes

.button, [type="text"], :hover - Specificity: 10

4. Elements, Pseudo-elements

p, ::before - Specificity: 1

Calculating Specificity Example:

p { color: red; } /* Specificity: 1 */

.intro { color: blue; } /* Specificity: 10 */

#main p { color: green; } /* Specificity: 101 */

/* #main p wins! (101 > 10 > 1) */

Preference of Rules (The Cascade)

CSS stands for "Cascading" Style Sheets. When multiple rules apply to an element, these factors determine which style wins:

Order of Importance:

  1. !important - Overrides everything (use sparingly!)
  2. Inline styles - style attribute in HTML
  3. Specificity - More specific selectors win
  4. Source order - Later rules override earlier ones
p {
  color: blue;
}

p {
  color: red;
}

/* Paragraph will be red (last rule wins) */

Chrome Dev Tools

Chrome DevTools is a set of web developer tools built into Google Chrome that helps you inspect, debug, and modify your CSS in real-time.

How to Open DevTools:

  • Right-click on any element → "Inspect"
  • Keyboard shortcuts:
    • Windows/Linux: F12 or Ctrl + Shift + I
    • Mac: Cmd + Option + I
  • Menu: Three dots → More tools → Developer tools

Key Features:

  • Elements Panel: View and edit HTML/CSS live
  • Styles Pane: See all CSS rules applied to an element
  • Computed Tab: See final computed styles
  • Box Model Viewer: Visualize margins, padding, borders
  • Color Picker: Modify colors interactively
  • Live Editing: Changes update instantly (temporary)

Adding Color

Colors in CSS can be specified in several ways:

Color Formats:

1. Color Names

color: red;
background-color: blue;
/* 140 named colors like red, blue, green, etc. */

2. Hexadecimal (Hex)

color: #FF0000; /* Red */
color: #00FF00; /* Green */
color: #0000FF; /* Blue */
color: #FFF; /* Short for #FFFFFF (white) */

3. RGB (Red, Green, Blue)

color: rgb(255, 0, 0); /* Red */
color: rgb(0, 255, 0); /* Green */
/* Values: 0-255 for each channel */

4. RGBA (RGB with Alpha/Opacity)

color: rgba(255, 0, 0, 0.5); /* 50% transparent red */
/* Alpha: 0 (transparent) to 1 (opaque) */

5. HSL (Hue, Saturation, Lightness)

color: hsl(0, 100%, 50%); /* Red */
color: hsla(120, 100%, 50%, 0.3); /* 30% transparent green */

CSS Units

CSS units specify the measurement of properties like width, height, font-size, etc.

Absolute Units (Fixed Size)

Unit Description
px Pixels - Most common, 1px = 1/96th of an inch
pt Points - 1pt = 1/72nd of an inch
cm, mm, in Centimeters, millimeters, inches

Relative Units (Responsive)

Unit Description
% Percentage - Relative to parent element
em Relative to font-size of the element (2em = 2 times)
rem Relative to root (html) font-size (more predictable)
vw Viewport width - 1vw = 1% of viewport width
vh Viewport height - 1vh = 1% of viewport height

How to Add Border to an Element

Borders add an outline around elements. They have three properties: width, style, and color.

Border Properties:

/* Individual properties */
border-width: 2px;
border-style: solid;
border-color: red;

/* Shorthand (recommended) */
border: 2px solid red;

Border Styles:

solid - Solid line
dashed - Dashed line
dotted - Dotted line
double - Double line

Individual Sides:

border-top: 1px solid black;
border-right: 2px dashed blue;
border-bottom: 3px dotted red;
border-left: 4px solid green;

/* Rounded corners */
border-radius: 10px;

Text Styling

CSS provides many properties to style and format text.

Font Properties:

font-family: Arial, sans-serif;
font-size: 16px;
font-weight: bold; /* or 100-900 */
font-style: italic;
font-variant: small-caps;

Text Properties:

color: #333;
text-align: center; /* left, right, center, justify */
text-decoration: underline; /* none, underline, line-through */
text-transform: uppercase; /* lowercase, capitalize */
line-height: 1.5; /* Space between lines */
letter-spacing: 2px; /* Space between characters */
word-spacing: 5px; /* Space between words */
text-shadow: 2px 2px 4px rgba(0,0,0,0.5);

Adding Background to an Element

Background properties control the background area of elements.

Background Properties:

/* Solid color */
background-color: #f0f0f0;

/* Background image */
background-image: url('image.jpg');
background-repeat: no-repeat; /* repeat, repeat-x, repeat-y */
background-position: center; /* top, bottom, left, right, center */
background-size: cover; /* contain, cover, 100px, 50% */
background-attachment: fixed; /* scroll, fixed */

/* Shorthand */
background: #f0f0f0 url('img.jpg') no-repeat center/cover;

Gradient Backgrounds:

/* Linear gradient */
background: linear-gradient(to right, red, yellow);

/* Radial gradient */
background: radial-gradient(circle, red, yellow);

The CSS Box Model

The Box Model is fundamental to CSS layout. Every HTML element is treated as a rectangular box.

Box Model Components (from inside out):

CONTENT
Padding
Border
Margin
  • Content: The actual content (text, images)
  • Padding: Space between content and border
  • Border: Line around padding
  • Margin: Space outside the border

Box-Sizing Property:

/* Default - padding and border add to width */
box-sizing: content-box;

/* Better - padding and border included in width */
box-sizing: border-box; /* ✅ Recommended */

Margins

Margins create space OUTSIDE an element's border, between elements.

Margin Properties:

/* Individual sides */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 10px;
margin-left: 20px;

/* Shorthand - 4 values (top, right, bottom, left) */
margin: 10px 20px 10px 20px;

/* Shorthand - 2 values (vertical, horizontal) */
margin: 10px 20px;

/* Shorthand - 1 value (all sides) */
margin: 10px;

/* Center element horizontally */
margin: 0 auto;

/* Negative margin (pull element) */
margin-top: -10px;

⚠️ Margin Collapse

When two vertical margins meet, they collapse into a single margin (the larger one). This doesn't happen with padding!

Padding

Padding creates space INSIDE an element's border, between content and border.

Padding Properties:

/* Individual sides */
padding-top: 10px;
padding-right: 20px;
padding-bottom: 10px;
padding-left: 20px;

/* Shorthand - 4 values (top, right, bottom, left) */
padding: 10px 20px 10px 20px;

/* Shorthand - 2 values (vertical, horizontal) */
padding: 10px 20px;

/* Shorthand - 1 value (all sides) */
padding: 10px;

Margin vs Padding:

Aspect Margin Padding
Location Outside border Inside border
Background No background color Has background color
Collapse Can collapse Never collapses
Negative values ✅ Allowed ❌ Not allowed

Display Property

The display property controls how an element is displayed and how it interacts with other elements.

Display Values:

block

display: block;
  • Takes full width available
  • Starts on a new line
  • Can set width and height
  • Examples: <div>, <p>, <h1>

inline

display: inline;
  • Only takes space needed
  • Stays on same line
  • Cannot set width/height
  • Examples: <span>, <a>, <strong>

inline-block

display: inline-block;
  • Stays on same line (like inline)
  • Can set width/height (like block)
  • Best of both worlds!

none

display: none;
  • Element is completely hidden
  • Takes no space
  • Different from visibility: hidden (which takes space)

flex

display: flex;
  • Creates flexible layout container
  • Modern layout system
  • Aligns and distributes space easily

grid

display: grid;
  • Creates grid layout container
  • 2D layout system (rows and columns)
  • Powerful for complex layouts

Position Property

The position property specifies how an element is positioned in the document.

Position Values:

static (default)

position: static;

Normal document flow. top, right, bottom, left properties have no effect.

relative

position: relative;
top: 10px;
left: 20px;

Positioned relative to its normal position. Original space is preserved.

absolute

position: absolute;
top: 0;
right: 0;

Positioned relative to nearest positioned ancestor. Removed from normal flow.

fixed

position: fixed;
top: 0;
left: 0;

Positioned relative to viewport. Stays in place when scrolling. Great for navbars!

sticky

position: sticky;
top: 0;

Hybrid of relative and fixed. Sticks when you scroll past it.

Z-Index (Layering):

z-index: 10; /* Higher values appear on top */
/* Only works with positioned elements (not static) */

Min/Max Width and Height

Control the minimum and maximum dimensions of elements for responsive design.

Width Properties:

width: 300px; /* Fixed width */
width: 50%; /* Relative to parent */
max-width: 800px; /* Won't grow beyond this */
min-width: 200px; /* Won't shrink below this */

Height Properties:

height: 200px; /* Fixed height */
height: 100vh; /* Full viewport height */
max-height: 500px; /* Won't grow beyond this */
min-height: 100px; /* Won't shrink below this */

💡 Pro Tip:

Use max-width instead of width for responsive layouts. The element will shrink on smaller screens!

Responsive Container Example:

.container {
  width: 100%;
  max-width: 1200px; /* Don't exceed 1200px */
  min-width: 320px; /* Minimum mobile width */
  margin: 0 auto; /* Center it */
  padding: 0 20px;
}

Key Takeaways

  • CSS separates content from presentation
  • Use external CSS for best maintainability
  • Understand the Box Model (content, padding, border, margin)
  • Master selectors and specificity for precise styling
  • Use relative units (rem, %, vw, vh) for responsive design
  • Chrome DevTools is your best friend for debugging CSS
  • Practice with display and position properties for layouts
  • min-width/max-width are essential for responsive design
