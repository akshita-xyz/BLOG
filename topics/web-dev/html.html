HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure and content of a webpage using tags and elements.

Not Case Sensitive
Creates Web Pages
.html / .htm

What Does HTML Stand For?

Hypertext Markup Language
HTML is a language that uses markup (tags) to create text with hyperlinks and structure for web pages. It defines the skeleton of every website you visit.

Tags & Elements

HTML uses tags to mark up content. Tags are enclosed in angle brackets and usually come in pairs: an opening tag and a closing tag.

The <html> Element: Root Container

The <html> element is the root element that contains all other HTML elements. It wraps the entire HTML document and tells the browser "this is an HTML page."

The <div> Tag - Division/Section

The <div> tag defines a division or section in an HTML document. It's a container element used to group other HTML elements together.

Why Use <div> Tags?

✓ Group related content together
✓ Apply styles to multiple elements at once
✓ Create layout structures and sections
✓ Organize and structure your web page

Code

<div>
  <h2>Section Title</h2>
  <p>This is a paragraph inside a div.</p>
  <p>Another paragraph in the same div.</p>
</div>

Output

Section Title

This is a paragraph inside a div.

Another paragraph in the same div.

Note: The <div> tag is a block-level element, meaning it starts on a new line and takes up the full width available. It's often used with CSS classes or IDs to style specific sections of a page.

Basic HTML Document Structure

1 <!DOCTYPE html>
2 <html>
3 <head>
4 <title>Page Title</title>
5 </head>
6 <body>
7 <h1>Hello World</h1>
8 </body>
9 </html>
Try Different Tags:

Attributes

Attributes provide extra information about HTML elements. They are always applied within the start tag (opening tag) and usually come in name/value pairs like attribute="value"

Attribute Syntax

<a href="https://example.com">Link text</a>

Common Attributes

  • href — specifies the URL for links
  • src — specifies the source for images
  • alt — provides alternative text for images
  • id — unique identifier for an element
  • class — specifies one or more class names

HTML provides six levels of heading tags, from <h1> to <h6>. They define headings of different importance and size.

The <body> Tag

The <body> tag contains all the visible content of your webpage — text, images, links, buttons, etc.

The <body> tag can be used only ONCE in an HTML document. Every HTML page has exactly one body element that wraps all the visible content.

The <br> tag inserts a single line break (space). It's an empty tag, meaning it has no closing tag.

<p>Hello&nbsp;&nbsp;&nbsp;World</p>

<p>Hello&emsp;World</p>

Lists

HTML provides different types of lists to organize and structure content.

Unordered List

An unordered list starts with the <ul> tag. Each list item starts with <li>. Items are marked with bullets and have no special/specific order or sequence.

Code

<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ul>

Output

  • Coffee
  • Tea
  • Milk

List Type Attribute

You can change the bullet style using the type attribute.

<ul type="disc">
  • Item 1
  • Item 2
<ul type="circle">
  • Item 1
  • Item 2
<ul type="square">
  • Item 1
  • Item 2

Ordered List

An ordered list starts with the <ol> tag. Each list item starts with <li>. Items have a definite sequence and are marked with numbers.

Code

<ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>

Output

  1. First step
  2. Second step
  3. Third step

Digital Assets

HTML allows you to embed digital assets like images and videos directly into your web pages.

Images

The <img> tag is used to embed images. It's an empty tag (no closing tag) and uses the src attribute to specify the image path.

Code

Output

width — Sets the width of the image (in pixels)
height — Sets the height of the image (in pixels)
align — Aligns the image (left, right, middle, top, bottom)
border — Specifies the border thickness (in pixels)

Example with Size

<img src="photo.jpg"
     alt="My Photo"
     width="200"
     height="150">

Output

Anchor Tag <a> - Creating Links

The <a> (anchor) tag creates hyperlinks, which allow users to navigate from one page to another. Hyperlinks are the foundation of the web and connect web pages together.
Links dont have to be text only they can be image or any other HTML element.

Basic Syntax

<a href="url">Link Text</a>

External Links (Absolute URL)

External links point to pages on other websites. They use absolute URLs which include the full web address starting with http:// or https://.

Code

<a href="https://www.google.com">
  Visit Google
</a>

Output

Visit Google

Internal Links (Local Link)

Internal links point to pages within the same website. They use relative URLs or local links which show the path relative to the current page.

Code

<a href="about.html">
  About Us
</a>

Output

About Us

Same Page Links (Anchor Links)

Links that jump to a specific section on the same page. Use # followed by the element's id.

Code

<a href="#section1">
  Go to Section 1
</a>

<h2 id="section1">Section 1</h2>

Output

Go to Section 1

(Jumps to the element with id="section1")

Target Attribute

The target attribute specifies where to open the linked document.

target="_blank" — Opens in a new tab/window
<a href="https://example.com" target="_blank">Open in New Tab</a>
target="_self" — Opens in the same tab (default)
<a href="page.html" target="_self">Open in Same Tab</a>
target="_parent" — Opens in the parent frame
target="_top" — Opens in the full body of the window

Summary of Link Types

Absolute URL: Full web address → https://www.example.com/page.html
Relative URL: Path from current location → about.html or ../images/photo.jpg
Anchor Link: Same page section → #section-id

Element Types

HTML elements are categorized into different types based on how they behave and display on a web page.

Block-Level Elements

Block-level elements always start on a new line and take up the full width available (stretches from left to right). They create a "block" of content. Broswer automatically adds space before and after them.

Common Block-Level Elements:

<div> — Container for grouping content
<p> — Paragraph
<h1> - <h6> — Headings
<ul>, <ol>, <li> — Lists and list items
<form> — Form container
<table> — Table
<section>, <header>, <footer> — Semantic sections

Code

<div>Block 1</div>
<div>Block 2</div>
<p>Paragraph block</p>

Output

Block 1
Block 2
Paragraph block

(Each takes full width and starts on new line)

Inline Elements

Inline elements do not start on a new line and only take up as much width as necessary. They flow within the text content.

Common Inline Elements:

<span> — Generic inline container
<a> — Hyperlink
<img> — Image
<strong>, <b> — Bold text
<em>, <i> — Italic text
<code> — Code snippet
<br> — Line break

Code

<p>
  This is <strong>bold</strong> and
  this is <em>italic</em> text.
</p>

Output

This is bold and this is italic text.

(Inline elements flow within the text)

Semantic Elements

Semantic elements clearly describe their meaning to both the browser and the developer. They make the HTML more readable and accessible.

Common Semantic Elements:

<header> — Header section of page or article
<nav> — Navigation links
<main> — Main content of the document
<article> — Self-contained content (blog post, news article)
<section> — Thematic grouping of content
<aside> — Content aside from main content (sidebar)
<footer> — Footer section of page or article
<form> — Form for user input
<table> — Table with rows and columns of data

Example: Article

<article>
  <header>
    <h2>Article Title</h2>
  </header>
  <p>Article content...</p>

</article>

Output

Article content...

Author info

Why Use Semantic Elements? They improve SEO, accessibility for screen readers, and make your code more readable and maintainable.

Non-Semantic Elements

Non-semantic elements tell nothing about their content. They are generic containers used for styling and layout purposes without conveying meaning.

Common Non-Semantic Elements:

Form Validation

Form validation is the process of checking that a form has been filled in correctly before it is processed. It helps ensure that users provide data in the correct format and that required fields are not left empty. Validation improves user experience and data quality.

Types of Form Validation

There are two main types of form validation:

1. Client-Side Validation

Client-side validation occurs in the browser before the form data is sent to the server. It provides instant feedback to users, improving user experience and reducing server load.

Advantages:
  • Immediate feedback to users
  • Reduces server load and bandwidth usage
  • Better user experience (no page reload needed)
  • Can be implemented using HTML5 attributes or JavaScript
Disadvantages:
  • Can be bypassed by users (not secure)
  • May not work if JavaScript is disabled
  • Should always be combined with server-side validation
HTML5 Validation Attributes

HTML5 provides built-in validation attributes that can be added to form elements:

Attribute Description
required Specifies that an input field must be filled out before submitting
pattern Specifies a regular expression that the input must match
min / max Specifies minimum and maximum values for numeric inputs
minlength / maxlength Specifies minimum and maximum length of text input
type Validates based on input type (email, url, number, etc.)
Client-Side Validation Example

Code

<form>
  <label>Email:</label><br>
  <input type="email" name="email"
    required placeholder="user@example.com"><br><br>

  <label>Password (min 8 chars):</label><br>
  <input type="password" name="pwd"
    required minlength="8"><br><br>

  <label>Age (18-100):</label><br>
  <input type="number" name="age"
    min="18" max="100" required><br><br>

  <input type="submit" value="Submit">
</form>

Output

Try submitting without filling fields to see validation in action!

2. Server-Side Validation

Server-side validation occurs on the server after the form data has been submitted. This is the most secure form of validation as it cannot be bypassed by users.

Advantages:
  • More secure - cannot be bypassed by users
  • Works regardless of browser capabilities
  • Can perform complex validation (database checks, business logic)
  • Essential for security and data integrity
Disadvantages:
  • Slower feedback (requires server round-trip)
  • Increases server load
  • May require page reload to show errors
  • Uses more bandwidth
Common Server-Side Validation Checks:
  • Data Type Validation: Ensuring data is of the correct type (string, number, date, etc.)
  • Format Validation: Checking if data follows specific patterns (email format, phone numbers)
  • Range Validation: Verifying values fall within acceptable ranges
  • Uniqueness Checks: Ensuring values are unique in the database (usernames, emails)
  • Business Logic Validation: Checking against business rules and constraints
  • Security Validation: Preventing SQL injection, XSS attacks, etc.

Best Practices

  • Use Both: Always implement both client-side and server-side validation for optimal security and user experience
  • Clear Error Messages: Provide specific, helpful error messages that guide users to fix issues
  • Inline Validation: Show validation feedback as users fill out forms, not just on submit
  • Visual Indicators: Use colors, icons, or borders to highlight valid/invalid fields
  • Validate Early: Check input format as soon as users leave a field
  • Never Trust Client Data: Always validate on the server, even if client validation passed

Favicon & Meta Tags

Favicon and meta tags are essential elements in the <head> section of an HTML document. They enhance user experience, improve SEO, and provide important information about your webpage to browsers and search engines.

Favicon

A favicon (short for "favorite icon") is a small icon that appears in the browser tab, bookmarks bar, and address bar. It helps users quickly identify your website among multiple open tabs.

What is a Favicon?

  • Small icon representing your website (typically 16x16 or 32x32 pixels)
  • Appears in browser tabs, bookmarks, and history
  • Common formats: .ico, .png, .svg, .gif
  • Improves brand recognition and user experience

Adding a Favicon

Place the favicon link in the <head> section of your HTML document:

Basic Favicon (ICO format)
<head>
  <link rel="icon" type="image/x-icon" href="/favicon.ico">
</head>
PNG Favicon (Recommended)
<head>
  <link rel="icon" type="image/png" href="/favicon.png">
</head>
SVG Favicon (Modern)
<head>
  <link rel="icon" type="image/svg+xml" href="/favicon.svg">
</head>
Multiple Sizes (Best Practice)
<head>
  <link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
  <link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
  <link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
</head>

The apple-touch-icon is used when users add your site to their iOS home screen.

Meta Tags

Meta tags provide metadata about the HTML document. They are placed in the <head> section and are not displayed on the page but are read by browsers, search engines, and social media platforms.

1. Essential Meta Tags

Character Encoding
<meta charset="UTF-8">

Specifies the character encoding for the document. UTF-8 supports all languages and special characters.

Viewport (Responsive Design)
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Essential for responsive design. Controls how the page is displayed on mobile devices.

Page Description
<meta name="description" content="Learn HTML basics, forms, validation and more. Complete guide to web development.">

Provides a brief description (150-160 characters) that appears in search engine results. Crucial for SEO.

Keywords
<meta name="keywords" content="HTML, web development, forms, validation, tutorial">

Lists relevant keywords for search engines. Note: Most modern search engines give this low priority.

Author
<meta name="author" content="Your Name">

Specifies the author of the document.

2. SEO Meta Tags

Robots
<meta name="robots" content="index, follow">

Controls how search engines crawl and index your page:
• index - Allow indexing
• noindex - Prevent indexing
• follow - Follow links
• nofollow - Don't follow links

Canonical URL
<link rel="canonical" href="https://www.example.com/page">

Specifies the preferred URL for duplicate content, helping prevent SEO issues.

3. Open Graph Meta Tags (Social Media)

Open Graph tags control how your page appears when shared on social media platforms like Facebook, LinkedIn, and Twitter.

<meta property="og:title" content="Page Title">
<meta property="og:description" content="Page description here">
<meta property="og:image" content="https://example.com/image.jpg">
<meta property="og:url" content="https://example.com/page">
<meta property="og:type" content="website">
<meta property="og:site_name" content="Your Site Name">
Property Description
og:title Title that appears when shared
og:description Brief description of the page
og:image Image displayed in social media preview
og:url Canonical URL of the page
og:type Type of content (website, article, etc.)

4. Twitter Card Meta Tags

Twitter uses its own meta tags to control how links appear on the platform.

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:site" content="@yourusername">
<meta name="twitter:title" content="Page Title">
<meta name="twitter:description" content="Page description">
<meta name="twitter:image" content="https://example.com/image.jpg">

5. Other Useful Meta Tags

Refresh/Redirect
<meta http-equiv="refresh" content="30">
<!-- Refresh page every 30 seconds -->

<meta http-equiv="refresh" content="5; url=https://example.com">
<!-- Redirect after 5 seconds -->
Theme Color (Mobile Browsers)
<meta name="theme-color" content="#4285f4">

Sets the color of the browser's address bar on mobile devices.

Complete Example

Here's a complete example with favicon and meta tags in the <head> section:

<!DOCTYPE html>
<html lang="en">
<head>
  <!-- Essential Meta Tags -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Learn web development with our comprehensive tutorials">
  <meta name="keywords" content="HTML, CSS, JavaScript, web development">
  <meta name="author" content="Your Name">

  <!-- Favicon -->
  <link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
  <link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
  <link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">

  <!-- SEO -->
  <meta name="robots" content="index, follow">
  <link rel="canonical" href="https://www.example.com/">

  <!-- Open Graph (Social Media) -->
  <meta property="og:title" content="Web Development Tutorial">
  <meta property="og:description" content="Learn web development with our tutorials">
  <meta property="og:image" content="https://example.com/image.jpg">
  <meta property="og:url" content="https://example.com/">
  <meta property="og:type" content="website">

  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Web Development Tutorial">
  <meta name="twitter:description" content="Learn web development">

  <!-- Theme Color -->
  <meta name="theme-color" content="#4285f4">

  <title>Web Development Tutorial</title>
</head>
<body>
  <!-- Your content here -->
</body>
</html>

Best Practices

  • Favicon: Use PNG or SVG format for better quality; provide multiple sizes for different devices
  • Description: Keep meta descriptions between 150-160 characters for optimal display in search results
  • Images: Use high-quality images (1200x630px recommended) for Open Graph tags
  • Viewport: Always include viewport meta tag for responsive design
  • Testing: Test social media previews using Facebook's Sharing Debugger and Twitter's Card Validator
  • Unique Content: Use unique titles and descriptions for each page
