Types of Cryptography

There are three main types of cryptography: Symmetric Key Cryptography, Asymmetric Key Cryptography, and Hash Functions. Each serves different purposes in securing data.

1. Symmetric Key Cryptography

Symmetric Key Cryptography uses a single shared key for both encryption and decryption. The sender and receiver must both have the same secret key.

  • Fast and efficient ‚Äî great for encrypting large amounts of data
  • Key exchange problem ‚Äî how do you securely share the key?
  • Examples: DES, AES, 3DES, Blowfish

2. Asymmetric Key Cryptography

Asymmetric cryptography uses two different keys: a public key (shared with everyone) and a private key (kept secret). A sender's public key is used for encryption and a receiver's private key is used for decryption. Public keys and Private keys are different. Even if the public key is known by everyone the intended receiver can only decode it because he holds his private key.

  • Solves key exchange ‚Äî public key can be shared openly
  • Slower than symmetric encryption
  • Examples: RSA, ECC, Diffie-Hellman

How it works:

  1. üîë Generate a public/private key pair

  3. üîí Encrypt using the recipient's public key
  4. üîì Decrypt using your private key (only you have it)

Real-world use: When you visit an HTTPS website, asymmetric encryption is used first to securely exchange a symmetric key, then symmetric encryption is used for speed.

3. Hash Functions

Hash functions are one-way functions. They convert any input into a fixed-length output (hash), but you cannot reverse the process to get the original input.

  • No key required ‚Äî just the algorithm
  • Fixed output size ‚Äî regardless of input size
  • One-way ‚Äî cannot derive input from output
  • Uses: Password storage, data integrity, digital signatures

SHA-1 (Secure Hash Algorithm 1)

  • Produces a 160-bit (20-byte) hash value
  • Developed by NSA, published in 1995
  • Now considered broken ‚Äî don't use for security

SHA-256

  • Part of the SHA-2 family
  • Produces a 256-bit (32-byte) hash value
  • Widely used ‚Äî Bitcoin, SSL certificates, password hashing
Example
Input: "geeksforgeeks" SHA-256: f8d59362da74ffe833332dc20508f12de6da6a9298c98b3b42873e7298fced78

MD5 (Message Digest 5)

  • Produces a 128-bit (16-byte) hash value
  • Very fast, but not secure ‚Äî collisions found
  • Still used for checksums and non-security purposes
Example
Input: "geeksforgeeks" MD5: a6eb56f80be8a120436d6f1c9b8d87ca
