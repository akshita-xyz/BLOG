Working of Cryptography

Cryptography transforms readable data (plaintext) into an unreadable format (ciphertext) using mathematical algorithms and keys. The reverse process restores the original data.

How It Works

The process involves several key components:

1. Plaintext is created:

The original readable message or data.

2. The Algorithm

A mathematical procedure that transforms data. Modern algorithms like AES and RSA are chosen, in the above example, Caesar cipher is used.

3. The Key

A piece of information (usually a large number) that controls the transformation. Without the correct key, decryption is practically impossible even if you know the algorithm. In the above example, the key is 3. (This key controls how the encryption or hashing is performed).

Encryption converts plaintext â†’ ciphertext

The plaintext is converted into ciphertext with the help of the algorithm and the key. In the above example, "Hello" is converted into "jknnq".

Decryption converts ciphertext â†’ plaintext

The ciphertext is converted into plaintext with the help of the algorithm and the key. In the above example, "jknnq" is converted into "Hello".

Integrity checks may be used

Integrity checks may be used to ensure that the data has not been tampered with. In the above example, the integrity check is used to ensure that the data has not been tampered with. (This is done by using a hash function to create a hash of the data and then comparing it to a hash of the data that was received.).

4. Symmetric vs Asymmetric

  • Symmetric: Same key for encryption and decryption (like AES). Faster, but requires secure key exchange. Secure key exchange is the challenge in cryptography of securely sharing secret encryption keys between parties, especially in large networks, because sending key to unverified parties risks handing key to people who are not authorized to have it.
  • Asymmetric: Different keys - a public key to encrypt, a private key to decrypt (like RSA). Slower, but solves the key distribution problem. The key distribution problem is the challenge in cryptography of securely sharing secret encryption keys between parties, especially in large networks, because sending the key over an insecure channel risks interception, breaking the whole system

Real-World Example

When you visit a website with HTTPS:

  1. Your browser and the server exchange public keys
  2. They use asymmetric encryption to securely share a symmetric key
  3. All further communication uses the faster symmetric encryption
  4. Anyone intercepting the traffic sees only ciphertext
