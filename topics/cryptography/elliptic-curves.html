Elliptic Curve Cryptography

Elliptic Curve Cryptography (ECC) is a type of asymmetric encryption that provides strong security with smaller keys than RSA. It's used everywhere ‚Äî from HTTPS to Bitcoin to your phone's secure enclave.

What's an Elliptic Curve?

An elliptic curve is a mathematical curve defined by this equation:

y¬≤ = x¬≥ + ax + b
The general form of an elliptic curve equation

When plotted, it creates a smooth, symmetric curve. The most famous curve used in cryptography is secp256k1 (used by Bitcoin), where a=0 and b=7:

y¬≤ = x¬≥ + 7
The secp256k1 curve used in Bitcoin

The Elliptic Curve: y¬≤ = x¬≥ + 7

The curve is symmetric about the x-axis. For every point P, there's a mirror point -P

Key Properties of the Curve

1
Symmetry
The curve mirrors across the x-axis. If (x, y) is on the curve, so is (x, -y)
2
Infinity Point
There's a special "point at infinity" (O) that acts like zero in addition
3
Line Intersection
Any line intersects the curve at exactly 3 points (counting infinity)

Point Addition: The Magic Operation

The key to ECC is point addition. To add points P and Q on the curve:

Point Addition: P + Q = R

Draw a line through P and Q, find where it hits the curve again, then flip vertically

Point Addition Steps:

  1. Draw a line through points P and Q
  2. Find where this line intersects the curve a third time (-R)
  3. Reflect that point across the x-axis to get R
  4. Result: P + Q = R

Point Multiplication (Scalar Multiplication)

Point multiplication is just repeated addition:

k √ó P = P + P + P + ... (k times)
Multiply point P by scalar k

This is easy to compute forward (even for huge k, using clever shortcuts), but nearly impossible to reverse. Given the result R = k √ó P, you can't figure out k even if you know P and R. This is the Elliptic Curve Discrete Logarithm Problem (ECDLP).

How ECC Creates Keys

1
Pick a Curve
Choose a standard curve (like secp256k1) with known generator point G
2
Generate Private Key
Pick a random large number k (your private key ‚Äî keep it secret!)
3
Compute Public Key
Calculate K = k √ó G (your public key ‚Äî share it freely)
Public Key = Private Key √ó G
Easy to compute, impossible to reverse

Why Brute Force Won't Work

Brute force is a trial-and-error method, especially in computing, that involves systematically trying every possible combination to crack passwords, encryption keys, or find data, relying on sheer computational power rather than cleverness.

You might wonder: why not just guess the private key? Let's see why that's absolutely impossible.
